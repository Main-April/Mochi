# Refactoring Guide

Propose des améliorations en suivant ces priorités :

1. **Éliminer la duplication** (DRY)
2. **Simplifier la complexité** (noms clairs, fonctions courtes)
3. **Extraire la logique** (séparer responsabilités)
4. **Typage robuste** (ajouter des types si pertinent)

Pour chaque suggestion :
- Montrer l'avant/après
- Expliquer le gain en une phrase
- Évaluer le risque de régression (faible/moyen/élevé)
